# Що поїсти? 🍽 — сімейний Telegram-бот з базою рецептів

Бот веде **приватну базу рецептів** для вашої групи (сім'ї). Надішліть рецепт
у будь-якому вигляді — фото, скрін, PDF, текст, пересланий пост — бот розпізнає
його (Gemini через OpenRouter), запропонує категорії, а далі відповідає на
запити на кшталт «що на вечерю?», «меню на тиждень до 2000 ккал» **виключно зі
збережених рецептів**, нічого не вигадуючи.

## Можливості

- 📷 Розпізнавання рецептів з фото/скрінів/PDF/тексту (укр/рос/англ)
- 🗂 Категорії: сніданок, обід, вечеря, десерт, салати, дієта, загальна (+ складність 1–4)
- 👨‍👩‍👧 Групи: спільна база рецептів за invite-посиланням, листування у кожного своє
- 🍽 Підбір страв: «що з курки?», «до чаю не солодке», «обіди з яловичиною»
- 📅 Меню на день/тиждень з балансуванням, калоріями і правилом «страва не частіше 1 разу на тиждень»
- 🛒 Агрегований список покупок, який можна переслати
- 🔒 Whitelist доступу — бот приватний (fail-closed: порожній `ALLOWED_USER_IDS`
  нікого не пускає; відкритий доступ — лише явним `ALLOWED_USER_IDS=*`)

**Відоме обмеження:** кількість персон передається AI при підборі меню, але
кількості інгредієнтів у списку покупок беруться з рецептів як є і на персон
не масштабуються (базові порції рецептів невідомі — вигадувати коефіцієнти
означало б порушити принцип нуль-галюцинацій).

## Налаштування

1. Створіть бота у [@BotFather](https://t.me/BotFather) → отримайте `BOT_TOKEN`.
2. Отримайте ключ AI (одне з двох):
   - **Рекомендовано:** OpenRouter (платний, стабільний) — https://openrouter.ai/keys
     → `OPENROUTER_API_KEY`. Модель — `OPENROUTER_MODEL` у `.env`
     (зараз `google/gemini-3.1-flash-lite-preview`); міняється без правок коду.
   - Запасний варіант: безкоштовний ключ Gemini API — https://aistudio.google.com/apikey
     → `GEMINI_API_KEY` (безкоштовний tier має ~10 запитів/хв).
   За наявності обох пріоритет має OpenRouter.
3. Дізнайтесь свій Telegram id (напр. у [@userinfobot](https://t.me/userinfobot)).
4. `cp .env.example .env` і заповніть значення.

## Запуск локально (для розробки)

Потрібен [uv](https://docs.astral.sh/uv/):

```bash
uv sync --extra dev      # створить .venv і поставить залежності (Python 3.12 підтягне сам)
uv run python -m bot.main
```

Тести: `uv run pytest`

## Запуск у Docker (на сервері)

```bash
docker compose up -d --build
docker compose logs -f bot   # подивитись логи
```

База SQLite лежить у `./data/bot.db` — бекап це просто копія цього файлу.

### Очистити базу з нуля

```bash
docker compose down
cp data/bot.db "data/bot.db.bak-$(date +%Y%m%d-%H%M)"   # бекап про всяк
rm data/bot.db
docker compose up -d --build                            # міграції створять порожню базу
```

Порядок дій, щоб рецепти одразу були спільними: `/start` → «👨‍👩‍👧 Група» →
«➕ Створити групу» і зробити її активною → **потім** заливати рецепти →
надіслати рідним invite-посилання. Якщо створити групу вже після додавання
рецептів — бот запропонує перенести їх у нову групу.

## Пам'ятка для сервера з 1 ГБ RAM (Oracle Free Tier)

```bash
# 2 ГБ swap як страховка
sudo fallocate -l 2G /swapfile && sudo chmod 600 /swapfile
sudo mkswap /swapfile && sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

## Архітектура (коротко)

Правила шарів і межі відповідальності: [ARCHITECTURE.md](ARCHITECTURE.md)
(перевіряються автоматично тестами `tests/test_architecture.py`).

- `bot/handlers/` — Telegram-хендлери (aiogram 3, long polling)
- `bot/services/llm/` — абстракція LLM (`base.py`) + реалізації: OpenRouter
  (`openrouter.py`, основна) і Gemini API (`gemini.py`, fallback); спільні
  ретраї з класифікацією помилок у `retry.py`. Замінити провайдера = написати
  один клас.
- `bot/services/` — підбір страв, планувальник меню, список покупок
- `bot/db/` — SQLAlchemy-моделі і репозиторій; міграції — Alembic
- **Принцип нуль-галюцинацій:** LLM лише (1) екстрактує структуру з вхідних даних
  і (2) обирає ID рецептів зі списку кандидатів. Текст рецепта завжди рендериться
  дослівно з БД.
